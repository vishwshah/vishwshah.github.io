LegalFD

LegalFD is one of my favorite and ambitious project. I have used my creativity and technique at the most in this project.

LegalFD was my capstone project for our client Sam Garcea. Sam is a lawyer who practices mainly in family court matters. He wanted to create a system for the society where if a person what to reperesent himself in a court he can use this website to create Legal Document and Legal Forms for their case. Basically according to the Judicial System, if they go to lawyer for help they have to pay Legal Fees (which is expensive).




We were a team of 4 members. Created a web application using Java as our language with Spring-Boot Framework for back-end and Bootstrap for front-end.

We have used Amazon Web Services for hosting and deploying our website. Let me give you overall briefs about the web application:

  • Authentication and Security
  • EC2 Services from AWS
  • Relational Database
  • Mailing services



There are three different sides of this website, Admin, Client and Lawyer. Follwoing are the features of each sides:

  • CRUD Operations
  • Uploading documents
  • Editing and Deleting User
  • Mailing
  • Third-Party Payment

We have used multiple types of different AWS services. EC2 is an instance where we store the documents uploaded by the client, lawyer, and admin (directory structure). Relational Database to store information about the client as well as credentials. We have BCrypted the password for security purposes as well.

Overall the experience was really amazing it was the first time I ever worked for a client in the real world. I hope to develop this kind of web application using different languages.
